Ygeiopolis Healthcare System - Improvements To Do

Team planning document for the final database submission.
The core schema is now strong: constraints, triggers, stored procedures, validation checks, portable setup scripts, and a larger data generator are in place. The remaining work is mostly final-submission packaging, query writing, evidence gathering, and report polishing.

1. Highest Priority Before Submission

Priority Task Why It Matters Expected Output
P0 Create the final SQL query files for Q01 to Q15. The assignment grades the query section heavily. Each query must return valid rows, otherwise it will not be graded. sql/Q01.sql through sql/Q15.sql.
P0 Run every query against a freshly loaded database and save output files. The deliverable explicitly asks for one output text file per query. sql/Q01_out.txt through sql/Q15_out.txt.
P0 Prepare the Q4 and Q6 optimization evidence. Q4 and Q6 require EXPLAIN or EXPLAIN ANALYZE, an alternative FORCE INDEX or hint version, cost/runtime comparison, and a short conclusion. Report section with screenshots or copied plans, plus the two SQL variants.
P0 Confirm the official reference data story. The brief says ICD-10, KEN, drugs, and medical procedure reference data should come from official sources, not invented names/codes. A short table in the report listing official, cleaned, fallback, and synthetic data.
P0 Create final diagrams in the exact required format. The final zip requires ER and relational diagrams as PDFs. diagrams/er.pdf and diagrams/relational.pdf.
