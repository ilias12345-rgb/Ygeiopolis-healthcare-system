# SQL Guide

This folder contains the database-facing part of the project.

## Files

- `schema.sql`: creates the `yg_eupolis_hospital` database, tables, constraints, indexes, and triggers.
- `load_workbench_absolute.sql`: loads generated CSV data from the absolute local paths used in MySQL Workbench during the exercise.
- `validation.sql`: sanity checks to run after loading data.
- `load.sql`: generated by `scripts/generate_data.py` inside a data bundle. It uses relative CSV paths and is better for reproducible runs.

## Execution Order

1. Run `schema.sql`.
2. Enable `LOCAL INFILE` in MySQL/MySQL Workbench.
3. Load data with either the generated relative loader or the existing Workbench absolute loader.

Example:

```bash
mysql -u root -p < sql/schema.sql
mysql --local-infile=1 -u root -p < hospital_dataset_bundle/sql/load.sql
mysql -u root -p yg_eupolis_hospital < sql/validation.sql
```

## Schema Areas

- Organization: departments, beds, operating/procedure rooms.
- Personnel: parent `personnel` table plus doctor/nurse/admin specializations.
- Scheduling: department shifts and shift assignments.
- Patient care: emergency visits and hospitalizations.
- Clinical data: ICD-10 diagnoses, KEN costing, procedures, lab tests, prescriptions, allergies.
- Support data: evaluations and image metadata.

## Business Rules In SQL

The schema intentionally uses triggers for rules that are difficult to express with simple constraints:

- doctor supervision hierarchy validation;
- prescription allergy prevention;
- procedure room and staff overlap prevention;
- procedure room type validation;
- shift supervision, monthly limits, rest time, and night-shift limits;
- automatic hospitalization cost calculation from KEN values.

## Notes

The absolute loader is useful for your current machine only. For another machine or a final submission, prefer generating a fresh bundle with `scripts/generate_data.py`, then use the generated `hospital_dataset_bundle/sql/load.sql`.
